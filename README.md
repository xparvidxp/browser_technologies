# Week 1 voorbij

In de eerste week heb ik gewerkt aan het opzetten en vormgeven van de eerste vragenlijst voor de aangifte erfbelasting. Ik ben begonnen met het structureren van het formulier in HTML en heb geleerd hoe je een formulier overzichtelijk kunt opbouwen met elementen zoals `<form>`, `<fieldset>`, `<legend>` en `<label>`. Daarna heb ik mij vooral gefocust op het stylen van de pagina zodat deze visueel lijkt op de website van de NS. Tijdens sprint-0 heb ik geleerd dat het beter is om eerst voor een klein scherm te ontwerpen en daarna op te schalen naar grotere schermen (mobile-first). Het stylen heeft het grootste deel van de tijd gekost, maar uiteindelijk ben ik tevreden met het resultaat. Daarnaast heb ik een artikel gelezen over het belang van consistente stijlgidsen en goed UX/UI-design, waarin werd uitgelegd dat iconen alleen moeten worden gebruikt wanneer ze een duidelijke functionaliteit ondersteunen.


# Dag 3 - Maandag 2 Maart

Ik heb vandaag gewerkt aan de 2e pattern, dat is dat je bank- en spaarrekeningen moet aangeven. Bovenaan kan je het rekeningtype selecteren, gezamenlijk of privé. Als je er meer hebt kun je extra velden toevoegen door op een knop te drukken. Ik heb geleerd hoe ik dit met JS kan doen.
Ik heb ervoor gekozen om dit met JS te doen, omdat JS een taal is waar ik geen ervaring mee heb, en graag iets meer over wil leren.
Morgen wil ik verder met deze pattern af te maken en aan de eerste pattern beginnen. Hier ben ik totaal 3 uur mee bezig geweest.

# Dag 4 - Dinsdag 3 Maart

Ik ben vandaag verder gegaan aan de pattern die bankrekeningen toevoegt, en verwijdert. En begonnen aan de pattern die vragen verbergt zodra ze niet relevant zijn. Ik heb dit geheel zonder JS gedaan. Hier ben ik ongeveer 4 uur mee bezig geweest. De volgende keer ga ik verder met progressive enhancement, en validatie.

# Week 2 voorbij

In de tweede week heb ik gewerkt aan interactieve patronen in het formulier. Ik heb een pattern gemaakt waarmee gebruikers bank- en spaarrekeningen kunnen aangeven en waarbij extra rekeningen toegevoegd of verwijderd kunnen worden met knoppen. Hiervoor heb ik JavaScript gebruikt, omdat ik nog weinig ervaring heb met deze taal en hier graag meer over wilde leren. Daarnaast ben ik begonnen met een pattern waarbij vragen automatisch verborgen worden wanneer ze niet meer relevant zijn op basis van eerdere antwoorden. Dit heb ik eerst zonder JavaScript opgelost met CSS. In totaal heb ik hier ongeveer 7 uur aan gewerkt. In de volgende fase wil ik verder gaan met progressive enhancement en het toevoegen van validatie aan het formulier.


# Dag 5 - Maandag 9 Maart

Vandaag heb ik verder gewerkt aan het formulier voor de aangifte erfbelasting. Ik heb mij vooral gefocust op het laten werken van het formulier met en zonder JavaScript. Ik heb ervoor gezorgd dat extra bankrekeningen zichtbaar blijven wanneer JavaScript uitstaat, maar automatisch verborgen worden wanneer JavaScript actief is. Tegelijkertijd heb ik de knoppen voor het toevoegen en verwijderen van rekeningen zo aangepast dat deze alleen zichtbaar zijn wanneer JavaScript werkt. Daarnaast heb ik verschillende HTML-validaties toegevoegd, zoals patterns voor namen en IBAN-nummers. In totaal heb ik hier ongeveer 4 uur aan gewerkt. Vandaag heb ik geleerd hoe progressive enhancement werkt en hoe CSS-selectors en HTML-validatie beter samenwerken. Morgen wil ik verder werken aan het verbeteren van de validatie.
